st 运行 NeoVim 问题

最近在配置 NeoVim，其中用到了一个插件 hlchunk.nvim，我发现我配置之后的效果跟插件的效果完全不同

这是作者的效果

而这是我的效果

我的效果的那条线是断层的，这是什么原因导致的？

觉得是终端的问题，我用的是 st 终端，我尝试换了一个终端发现确实没上面的问题了，那就是 st 终端的问题了

最后定位到 st 的config.h配置文件的一个变量，将st-256color更改为xterm-256color解决上面问题

1
2
3

/* default TERM value */
// char *termname = "st-256color";
char *termname = "xterm-256color";

然后我又尝试就配置 Alacritty，又发现了一个新问题

在 Tmux 下打开 NeoVim 会发现主题颜色没效果

正常效果（非 Tmux 环境下运行 NeoVim）如图

异常效果（Tmux 环境下运行 NeoVim）如图，而且切换主题背景色基本没有什么变化

起初以为是 Tmux 没有设置真彩色，设置真彩色见下，然后发现并没有解决这个问题

最后排查发现是 Alacritty 没有设置真彩色，设置真彩色见下，至此问题解决

给Tmux设置真彩色

tmux > 2.2 后开始支持真彩色。在 .tmux.conf 中添加如下内容：

1
2

set -g default-terminal screen-256color
set-option -ga terminal-overrides ",*256col*:Tc" # 这句是关键

1
2

# 终端里执行脚本
curl -fL https://raw.githubusercontent.com/Beavan/tools/master/sh/24-bit-color.sh |bash

没有设置真彩色

设置了真彩色

Alacritty 设置真彩色

1
2

[env]
TERM = "xterm-256color"
